这个项目是一个基于 Bore 内核定制开发的 可视化内网穿透工具。它将复杂的命令行操作封装进一个简洁的图形界面（GUI）中，特别针对“单文件化”和“跨架构兼容性”进行了深度优化。

以下是该项目的几个核心特点：

1. 技术核心：Bore
Bore 是一个用 Rust 编写的高性能、极简反向代理工具。它的设计初衷是“无配置”，允许你通过一个简单的命令将本地 TCP 端口（如 Web 开发环境、数据库、游戏服务器）映射到公网。

公共节点: 默认使用 bore.pub 作为中转服务器，用户无需自备服务器。

安全性: 不保存数据，仅做流量转发。

2. 项目亮点
架构自适应 (Auto-Arch): 程序会自动检测你运行环境是 32 位还是 64 位系统。即使在 Windows 7 这种新旧交替的环境下，它也能精准调用最匹配的内核版本（i686 或 x86_64）。

全自动化生命周期:

运行即释放: 内核集成在主程序中，启动时自动解压到当前目录，确保运行稳定性。

退出即清理: 停止服务或关闭窗口时，程序会自动删除临时释放的内核文件，保持本地文件夹整洁，不留“垃圾”。

静默运行: 屏蔽了原本会弹出的黑色 CMD 窗口，所有的运行日志和报错信息都实时抓取并显示在 UI 界面的日志框内。

智能地址识别: 程序会实时监控内核输出，一旦公网域名生成，会自动拼凑成完整的 URL 并提供“一键打开浏览器”的功能。

3. 应用场景
临时演示: 快速把本地正在开发的网站发给客户或同事查看。

远程调试: 调试微信回调、Webhooks 等需要公网地址的场景。

技术平权: 专为追求简单、高效、免安装的用户设计，尤其对老旧系统的兼容性做了特殊处理。
